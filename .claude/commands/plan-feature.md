Plan the implementation of a new feature for this project. The user will describe the feature.

Do the following:
1. Read the relevant existing files to understand the current implementation.
2. Identify every file that needs to change.
3. For each file, describe exactly what needs to be added or modified — be specific about function names, data structures, and API contracts.
4. Identify any new data that needs to be scraped or indexed.
5. Estimate the effort: Low (1 tool + 1 frontend change), Medium (new data pipeline step), High (new infrastructure).
6. Flag any risks: breaking changes to the SSE protocol, CORS implications, Railway memory/CPU limits.

Present the plan clearly before writing any code. Wait for confirmation before implementing.

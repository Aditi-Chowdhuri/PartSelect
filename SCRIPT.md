# PartSelect Agent — Technical Summary

The agent is built on Anthropic's Claude claude-sonnet-4-6 model using native tool-use — no LangChain, no middleware, just a 40-line Python loop that sends each message to Claude alongside eight tool definitions, executes whichever tool Claude selects, appends the result back into the conversation, and repeats until Claude produces a final response. The eight tools cover every query pattern: symptom lookup, model compatibility, part detail retrieval, brand and type browsing, cart management, and semantic search — each one a thin data accessor over a local FAISS vector index and four pre-built JSON relational maps derived from 6,025 real parts scraped from the Internet Archive's Wayback Machine, because PartSelect's CDN blocks direct access. Retrieval is two-tier: structured queries hit the JSON maps in under a millisecond with no network call, while open-ended queries go through FAISS semantic search using all-MiniLM-L6-v2 embeddings in under 50 milliseconds, with a live Wayback proxy fallback only for single-part deep-dives. The backend is FastAPI streaming over server-sent events so every token, tool call, and cart update reaches the Next.js frontend in real time. The entire stack — vector index, relational maps, embeddings — runs on CPU with no external database, no Redis, no infrastructure beyond the Anthropic API key, which means the deployment footprint is a single Docker container and a Vercel frontend.
